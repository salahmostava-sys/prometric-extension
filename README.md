# Prometric Auto Register — Chrome Extension

> **الإصدار:** v3.0 · **المنصة:** Chrome / Chromium · **Manifest:** V3

إضافة Chrome تُؤتمت تسجيل المختبرين على موقع Prometric (IBTA MEA) بالكامل — سواء تسجيلاً فردياً أو دفعياً من Excel / CSV / Google Sheets أو عبر اللصق المباشر.

---

## 🌟 الميزات

| الميزة | التفاصيل |
|--------|---------|
| **وضع فردي (Single)** | أدخل الاسم والبريد الإلكتروني للبدء السريع |
| **وضع دفعي (Batch)** | ارفع ملف `.xlsx` أو `.csv` وسجّل المئات تلقائياً |
| **لصق الأسماء وتوليد الإيميلات** | الصق قائمة أسماء وسيقوم النظام بتوليد إيميلات لها فوراً (جديد) |
| **Google Sheets** | جلب البيانات مباشرةً من روابط Sheet مع فلتر للأيام |
| **متتبع الوقت (ETA Tracker)** | يعرض الوقت المتبقي لإنهاء دفعة التسجيل (جديد) |
| **توليد بيانات الدخول** | Username وPassword يُولَّدان تلقائياً مع تفادي الأسماء المحجوزة |
| **المنع من التكرار (Duplicate Guard)** | يتخطى الأسماء التي سُجلت مسبقاً لتوفير الوقت (جديد) |
| **إعادة المحاولة التلقائية** | يُعيد محاولة تسجيل العناصر الفاشلة بعد انتهاء الدفعة |
| **Turbo / Safe Mode** | تبديل سريع بين وضع السرعة ووضع الثبات |
| **Desktop Notifications** | إشعار على سطح المكتب عند انتهاء العملية |
| **السجل (History)** | سجل كامل لكل العمليات مع إمكانية تصديره (CSV) |

## 🛠 التثبيت

> ⚠️ **تحذير هام:** هذه الإضافة تستخدم نظام `require` وبالتالي تحتاج لعملية بناء (Build) قبل رفعها للمتصفح. يجب رفع المجلد المولد `dist` وليس المجلد الرئيسي لكي تعمل.

1. افتح موجه الأوامر في مجلد المشروع وشغل `npm install` ثم `npm run build`.
2. سيتم إنشاء مجلد جديد باسم `dist` يحتوي على الإضافة الجاهزة للعمل.
3. افتح متصفح Google Chrome واذهب إلى الرابط: `chrome://extensions/`
4. في أعلى يمين الصفحة، قم بتفعيل خيار **"وضع المطور" (Developer mode)**.
5. اضغط على الزر **"تحميل إضافة تم فك حزمها" (Load unpacked)**.
6. اختر مجلد **`dist`** (الموجود داخل مجلد المشروع).
7. ستظهر الأيقونة في شريط الإضافات (يُفضل تثبيتها للوصول السريع).

## 📖 دليل الاستخدام

### 1. التسجيل الفردي (Single)
يُستخدم لتسجيل شخص واحد فقط بسرعة:
1. افتح الإضافة واختر تبويب **Single**.
2. أدخل "الاسم الكامل" و "البريد الإلكتروني".
3. اضغط على **"Start Registration"**.

### 2. التسجيل الدفعي (Excel / CSV)
لتسجيل العشرات أو المئات دفعة واحدة:
1. جهّز ملف `.xlsx` أو `.csv` (بعمودي `Name` و `Email`). يمكنك تحميل النموذج عبر زر **"Download Template"**.
2. من تبويب **Excel / CSV**، اضغط لرفع الملف.
3. راجع ملخص الفحص (عدد الصفوف السليمة والأخطاء).
4. اضغط **"Start Batch Registration"**.

### 3. ميزة اللصق المباشر (Paste Names & Generate Emails) - *جديد*
إذا كان لديك أسماء فقط بدون إيميلات:
1. من تبويب Excel / CSV، اضغط على **"Paste Names & Generate Emails"**.
2. الصق قائمة الأسماء في النافذة.
3. حدد "نطاق الإيميل" (Email Domain) مثل `@example.com`.
4. اضغط **"Generate & Apply"** لتحويلها إلى جدول جاهز فوراً.

### 4. التسجيل عبر Google Sheets
1. من تبويب **Google Sheet**، الصق رابط الشيت (بصلاحية View).
2. اضغط **"Load Columns"** وحدد أعمدة الاسم والإيميل (والأيام اختيارياً).
3. اضغط **"Start Batch Registration"**.

## ⚙️ الإعدادات (Settings)

| الإعداد | الوصف |
|---------|-------|
| **Page Delay** | وقت الانتظار (بالثواني) بين كل خطوة وأخرى داخل صفحة Prometric |
| **Next User Delay** | وقت الاستراحة بين انتهاء مستخدم والبدء في الذي يليه |
| **Fully Automated** | الإنهاء التلقائي والضغط على Submit النهائي بدون تدخل منك |
| **Stability Mode** | انتظار أطول وإعادة محاولة في الصفحات البطيئة جداً |
| **Auto-Retry Failed** | إعادة محاولة تسجيل الأسماء الفاشلة بعد نهاية الطابور |
| **Password Pattern** | النمط المستخدم لتوليد كلمة المرور (شرح النمط بالأسفل) |

### توليد كلمات المرور (Password Pattern)
| Tag | المعنى | مثال على اسم Ahmed Hassan بنمط `{F}@{f}#$1970` |
|-----|--------|--------------------------------------------------|
| `{F}` | أول حرف من الاسم الأول (كبير) | `A` |
| `{f}` | أول حرف من الاسم الأول (صغير) | `a` |
| `{L}` | أول حرف من الاسم الأخير (كبير) | `H` |
| `{l}` | أول حرف من الاسم الأخير (صغير) | `h` |
> النتيجة للمثال أعلاه ستكون: `A@a#$1970`

### توليد أسماء المستخدمين (Usernames)
يُولَّد من أول كلمتين في الاسم (بالحروف الإنجليزية فقط): `"AHMED ALI" → AHMEDALI`. إذا كان محجوزاً، يُضاف إليه أرقام/حروف تلقائياً (`AHMEDALI1`, `AHMEDALI2`...).

## 📁 السجل (History)
- يحفظ كل عمليات التسجيل (الناجحة والفاشلة) متضمناً الأسماء، وكلمات المرور، وأسماء المستخدمين.
- يتيح لك تصدير السجل بالكامل كملف CSV بالضغط على **"Export (CSV)"**.
- تفادي التكرار (Duplicate Guard): الإضافة تتفقد هذا السجل لتجنب تسجيل أي شخص تم تسجيله سابقاً.

---

## 🖱 التحكم السريع (Context Menu)
اضغط **كليك يمين** على أيقونة الإضافة للوصول إلى:
- **Pause Registration:** إيقاف مؤقت للتسجيل.
- **Resume Registration:** استئناف التسجيل.
- **Stop & Clear Queue:** إيقاف تام ومسح باقي الطابور.

## 💻 هيكل الملفات والمعمارية للمطورين

```text
prometric-extension/
├── manifest.json       # إعدادات الإضافة (MV3)
├── background.js       # Service Worker — إدارة الطابور والرسائل
├── bridge.js           # وسيط بين ISOLATED world و MAIN world
├── content.js          # أتمتة الصفحات والتفاعل مع الـ DOM
├── popup.html / css / js # واجهة المستخدم (Control Panel)
└── icons...            # أيقونات الإضافة
```

**كيفية العمل:**
1. واجهة `popup.js` ترسل البيانات عبر `chrome.runtime.sendMessage` إلى `background.js`.
2. `background.js` يُدير طابور التسجيل (Queue) ويحفظ الحالة في `chrome.storage`.
3. `bridge.js` يستقرأ الحالة ويمررها إلى `content.js` للبدء بتعبئة البيانات.
4. `content.js` يستخدم `MutationObserver` لمراقبة التغييرات في الموقع والانتقال بين خطوات التسجيل.

## ⚠️ الأخطاء الشائعة والحلول

| الخطأ | السبب المحتمل | الحل |
|-------|--------------|------|
| `Username exhausted` | جميع احتمالات اسم المستخدم محجوزة | تغيير الاسم قليلاً في الملف |
| `Option IBTA MEA not found` | الصفحة لم تُحمَّل بالكامل | زيادة Page Delay |
| `Continue did not become ready` | بطء في استجابة الموقع | تفعيل Stability Mode |
| `Invalid email` | صيغة البريد خاطئة | مراجعة الملف وتصحيح الصيغة |
